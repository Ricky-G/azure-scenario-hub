How this works:

1. Each namespace has a StaticGatewayConfiguration that binds it to the gateway node pool.

2. AKS auto-provisions a Public IP Prefix (e.g. /28 = 16 IPs) for each namespace config.

3. Pods annotated with the gateway config egress through that prefix — the observed IP is within the prefix.

4. The prefix is not carved from the cluster VNet — it is a separate Azure Public IP Prefix resource.

This replicates OpenShift's EgressIP capability natively in AKS.
