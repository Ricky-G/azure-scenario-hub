# Community Code of Conduct

Azure Scenario Hub is a learning and engineering community. Participation should make the repository safer, clearer, and more useful for people with different backgrounds and levels of experience.

## Expected Behavior

- Be respectful, patient, and specific.
- Critique technical decisions and evidence, not people.
- Welcome questions asked in good faith and explain unfamiliar Azure or AI concepts without condescension.
- State assumptions, limitations, and uncertainty honestly.
- Protect credentials, personal information, tenant data, and unpublished security details.
- Respect licenses, attribution requirements, and the rights of others.
- Keep issues, discussions, reviews, and pull requests relevant to the repository.

## Unacceptable Behavior

- Harassment, discrimination, threats, intimidation, or personal attacks
- Sexualized language or unwanted attention
- Trolling, deliberate disruption, spam, or repeated bad-faith argument
- Publishing another person's private information or credentials
- Encouraging unsafe access to Azure resources, identities, data, or systems
- Publicly disclosing an uncoordinated security vulnerability in this repository
- Misrepresenting test results, authorship, licensing, or production readiness

## Maintainer Responsibilities

Maintainers may edit or remove comments, issues, discussions, pull requests, code, or other contributions that violate these expectations. They may warn, temporarily restrict, or permanently ban participants when necessary to protect the community and repository.

Moderation decisions should be proportionate, documented where appropriate, and focused on behavior in repository spaces.

## Reporting

- Report repository security vulnerabilities privately through [SECURITY.md](SECURITY.md).
- Use GitHub's content-reporting controls for abusive content.
- For threats, targeted harassment, or other serious platform-safety concerns, contact [GitHub Support](https://support.github.com/contact/report-abuse).

Do not include private information in a public conduct report. Good-faith reports and participants who raise concerns must not be retaliated against.

## Scope

This code applies to repository issues, discussions, pull requests, reviews, releases, and other spaces where a participant represents Azure Scenario Hub.
